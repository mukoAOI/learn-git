Linux 的信号系统是一种**进程间通信和异步事件通知机制**。它的核心作用在于允许操作系统内核、其他进程（在拥有足够权限的情况下）或进程自身**向目标进程发送一个简单的通知**，告知其发生了某个特定事件。进程收到信号后，可以采取预设的动作来响应这个事件。

信号系统的主要用途可以概括为以下几个方面：

1.  **进程控制与管理：**
    *   **终止进程：** `SIGTERM` (15) 是请求进程正常终止的标准信号。`SIGKILL` (9) 则是强制立即终止进程（该信号无法被进程捕获或忽略）。`SIGQUIT` (3) 通常由终端产生（如按 `Ctrl+\`），请求进程终止并生成核心转储文件。
    *   **暂停/恢复进程：** `SIGSTOP` (17,19,23) 强制暂停进程的执行。`SIGCONT` (18,19,25) 用于恢复被暂停的进程。这是作业控制的基础。
    *   **中断进程：** `SIGINT` (2) 是最常见的信号，通常由用户在终端按下 `Ctrl+C` 产生，请求进程中断（通常是正常终止，但进程可以选择自定义处理）。

2.  **异常与错误通知：**
    *   **硬件异常：** 当进程执行了非法操作（如访问无效内存地址、执行非法指令、浮点运算错误）时，内核会发送相应的信号（`SIGSEGV` - 段错误, `SIGILL` - 非法指令, `SIGFPE` - 浮点异常）。这通常是程序崩溃的原因。
    *   **软件条件触发：** 某些软件条件满足时也会发送信号，例如：
        *   子进程状态改变时（终止或暂停）向父进程发送 `SIGCHLD` (17)。
        *   当进程尝试写入一个读端已关闭的管道或 socket 时，会收到 `SIGPIPE` (13)。
        *   当进程设置的定时器（如 `alarm`, `setitimer`）到期时，会收到 `SIGALRM` (14)。
        *   尝试读写终端时收到后台作业的控制终端操作信号 `SIGTTIN` (21)/`SIGTTOU` (22)。

3.  **用户/系统事件通知：**
    *   终端窗口大小改变时，内核会向前台进程组发送 `SIGWINCH` (28)。
    *   用户按下 `Ctrl+Z` 时发送 `SIGTSTP` (20, 24, 26)（可捕获的停止信号，不同于 `SIGSTOP`）。
    *   进程尝试读取低速设备（如终端、管道、socket）但数据尚未准备好时，如果设置了非阻塞 I/O 或超时，可能会收到 `SIGIO` (29)/`SIGPOLL`（异步 I/O 可用）或 `SIGURG`（带外数据到达）。

4.  **自定义行为：**
    *   进程可以使用 `signal()` 或更健壮的 `sigaction()` 系统调用为大多数信号（除了 `SIGKILL` 和 `SIGSTOP`）注册**信号处理函数**。当信号到达时，进程会中断当前的控制流，转而执行该处理函数（这称为**信号处理**）。这允许进程：
        *   在收到 `SIGTERM` 或 `SIGINT` 时执行清理工作（关闭文件、释放资源、保存状态）再退出。
        *   忽略某些信号（如 `SIGCHLD`，避免僵尸进程，但通常更推荐显式 `wait`）。
        *   使用 `SIGALRM` 实现超时机制。
        *   捕获 `SIGSEGV` 或 `SIGFPE` 进行自定义错误日志记录或尝试恢复（虽然通常不建议尝试从严重错误中恢复）。
        *   响应 `SIGUSR1` (10) 和 `SIGUSR2` (12)（用户自定义信号）执行特定任务（如重新加载配置文件、切换日志文件、报告状态等）。

**信号的关键特性：**

*   **异步性：** 信号可以在进程执行的**任何时刻**到达（除了极少数原子操作）。
*   **轻量级：** 信号只传递一个数字（信号编号），不附带其他数据（实时信号 `SIGRTMIN` 到 `SIGRTMAX` 可以携带少量信息，但主要目的还是通知）。
*   **预定义类型：** Linux 定义了一系列标准信号（`kill -l` 查看列表），每个信号有唯一的编号和默认行为。
*   **处理方式：** 进程对信号的处理方式有三种：
    *   **默认行为：** 系统预定义的行为（如终止、终止并转储核心、忽略、暂停、继续）。
    *   **忽略：** 进程显式告知内核忽略该信号（`SIGKILL` 和 `SIGSTOP` 不可忽略）。
    *   **捕获：** 进程注册一个处理函数，信号到达时执行该函数。
*   **可靠性问题：** 标准信号可能会丢失（如果连续发送多个相同的标准信号，进程可能只收到一个）。实时信号解决了这个问题，支持排队。

**总结来说，Linux 信号系统是：**

*   **操作系统与进程沟通的桥梁：** 内核通知进程硬件错误、资源状态变化（如子进程结束）、用户输入（Ctrl+C）等关键事件。
*   **进程间协调的简单工具：** 一个进程可以请求另一个进程终止（`kill`）、暂停（`SIGSTOP`）、继续（`SIGCONT`）或执行自定义操作（`SIGUSR1/2`）。
*   **程序响应外部事件的手段：** 允许程序在运行时被中断以处理特定事件（超时、用户请求终止、配置文件更改通知等），实现更灵活的控制流。

理解信号机制对于编写健壮、可管理、能响应外部事件的 Linux 程序（尤其是守护进程、服务器程序、命令行工具）至关重要。
