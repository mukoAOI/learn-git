# kmp算法



KMP算法中的**前缀计算**（通常构建`next`数组或`prefix`表）是整个算法的核心预处理步骤。它的目的是在正式匹配开始前，**为模式串的每个位置预先计算出“当匹配失败时，模式串应该回退到哪个位置”的信息**，从而避免主匹配过程中不必要的回溯。

### 核心目标：计算部分匹配表（`next`数组）
*   **输入：** 模式串 `P` (长度为 `m`)
*   **输出：** 一个整数数组 `next[0..m-1]` (有时定义为 `next[0..m]`，索引含义略有不同，原理一致)
*   **`next[i]` 的含义：** 对于模式串 `P` 的前缀子串 `P[0..i]`，`next[i]` 的值等于该子串的 **最长相等真前缀和真后缀的长度**。
    *   **真前缀：** 不包含自身的前缀（例如 `"ABC"` 的真前缀有 `""`, `"A"`, `"AB"`）。
    *   **真后缀：** 不包含自身的后缀（例如 `"ABC"` 的真后缀有 `""`, `"C"`, `"BC"`）。
    *   **最长相等真前缀和真后缀：** 在 `P[0..i]` 的所有真前缀和真后缀中，找到最长的那个相等的子串，其长度就是 `next[i]`。

### 为什么需要这个表？
想象主串 `S` 和模式串 `P` 正在匹配。当匹配到 `S[i]` 和 `P[j]` 时发生不匹配：
1.  **暴力匹配：** 将 `P` 整体右移一位，`j` 重置为 `0`，`i` 回溯到之前匹配开始的下一位。效率低下（`O(n*m)`）。
2.  **KMP 优化：** 利用 `next` 表！
    *   不匹配发生在 `P[j]`，意味着 `P[0..j-1]` 已经和 `S` 的某个子串匹配成功了。
    *   `next[j-1]` 的值 `k` 告诉我们：`P[0..j-1]` 这个子串中，前 `k` 个字符 (`P[0..k-1]`) 和后 `k` 个字符 (`P[j-k..j-1]`) 是**相同**的。
    *   因此，我们可以直接将 `P` 向右滑动，**让 `P` 的前 `k` 个字符 (`P[0..k-1]`) 对齐到 `S` 上刚刚匹配成功的 `P[j-k..j-1]` 那个位置**。这相当于保持 `i` 不变（主串指针**不回退**），将 `j` 设置为 `k`（模式串指针回退到 `k`），继续比较 `S[i]` 和 `P[k]`。
    *   这个滑动的依据就是 `next[j-1] = k`。

### 前缀计算过程详解（构建 `next` 数组）
计算 `next` 数组的过程本身也是一个**字符串匹配**的过程，它是在模式串 `P` **自身**上进行的。我们使用两个指针：
*   `i`：当前正在计算 `next[i]` 的位置（后缀末尾位置）。
*   `j`：指向前缀的末尾位置，同时也代表了当前 `P[0..i-1]` 子串的最长相等真前缀和真后缀的长度（即 `next[i-1]` 的值）。

#### 算法步骤
1.  **初始化：**
    *   `next[0] = 0`：单个字符的子串没有真前缀和真后缀（或认为长度为0）。
    *   设置 `i = 1`（从模式串的第二个字符开始计算）。
    *   设置 `j = 0`（初始最长相等真前缀真后缀长度为0）。
2.  **循环（`i` 从 `1` 到 `m-1`）：**
    *   **情况1：`P[i] == P[j]`**
        *   匹配成功！后缀 `P[j]` 和前 `P[i]` 匹配上了。
        *   这意味着 `P[0..j]` 是 `P[0..i]` 的一个相等真前缀和真后缀。
        *   并且它的长度 `j+1` 是目前可能的最大长度（因为我们在之前的基础上延长了一位相等的字符）。
        *   所以 `next[i] = j + 1`。
        *   **更新指针：** `i = i + 1`, `j = j + 1` （继续比较下一个字符）。
    *   **情况2：`P[i] != P[j]`**
        *   匹配失败。
        *   **关键点：** 不能直接说 `next[i] = 0`！我们需要尝试**更短**的相等真前缀真后缀。
        *   `j` 当前指向的位置代表的是 `P[0..i-1]` 的最长相等真前缀真后缀的**长度**。当 `P[i]` 和 `P[j]` 不匹配时：
            *   如果 `j > 0`：利用 `next` 数组**已经计算好的信息**！将 `j` **回退**到 `next[j-1]` 的位置（相当于在 `P[0..j-1]` 这个子串上应用 KMP 思想，寻找一个更短的匹配前缀）。
            *   如果 `j == 0`：已经退无可退，说明 `P[0..i]` 没有任何非空的相等真前缀真后缀。`next[i] = 0`。
            *   **更新指针：** `i` **保持不变**（我们还在尝试计算 `next[i]`），`j` 更新为 `next[j-1]`（如果 `j>0`）或保持 `0`（如果 `j==0` 且还是不匹配）。
        *   **注意：** 在 `j` 回退之后，**回到循环开头**，再次尝试比较 `P[i]` 和 `P[j]`（此时的 `j` 已经更新了）。
3.  **结束：** 当 `i` 遍历完模式串 (`i == m`)，`next` 数组计算完成。

#### 图解示例：模式串 `P = "ABABC"`
| `i`  | `P[i]` | `j`  | `P[j]` | 比较 | 操作                                          | `next[i]` | 解释                                  |
| :--- | :----- | :--- | :----- | :--- | :-------------------------------------------- | :-------- | :------------------------------------ |
| 0    | A      | -    | -      | -    | -                                             | **0**     | 初始化                                |
| 1    | B      | 0    | A      | B≠A  | j>0? **j = next[j-1] = next[0] = 0** → 再比较 | **0**     | `j=0` 且 `P[1]≠P[0]` → `next[1]=0`    |
| 2    | A      | 0    | A      | A=A  | `next[2] = j+1 = 1`; i++, j++                 | **1**     | 匹配！`"ABA"` 中 `"A"` 是共同前后缀   |
| 3    | B      | 1    | B      | B=B  | `next[3] = j+1 = 2`; i++, j++                 | **2**     | 匹配！`"ABAB"` 中 `"AB"` 是共同前后缀 |
| 4    | C      | 2    | A      | C≠A  | j>0? **j = next[j-1] = next[1] = 0** → 再比较 | ?         | 不匹配，`j` 回退到 `next[1]=0`        |
| 4    | C      | 0    | A      | C≠A  | j>0? j=0 → `next[4] = 0`                      | **0**     | `j=0` 且 `P[4]≠P[0]` → `next[4]=0`    |

最终 `next = [0, 0, 1, 2, 0]`

### `next` 数组在 KMP 主匹配中的应用（简述）
假设主串 `S`，模式串 `P`，`next` 数组已计算。
1.  初始化主串指针 `i = 0`，模式串指针 `j = 0`。
2.  循环比较 `S[i]` 和 `P[j]`：
    *   如果 `S[i] == P[j]`：`i++`, `j++`。
    *   如果 `S[i] != P[j]`：
        *   如果 `j == 0`：说明模式串第一个字符就不匹配，`i++`（主串后移一位）。
        *   如果 `j > 0`：利用 `next` 数组！**`j = next[j-1]`**（模式串指针回退到 `next[j-1]` 的位置，主串指针 `i` **保持不动**）。
3.  如果 `j == m`（模式串匹配完成）：找到匹配，记录位置 `i - m`。
4.  继续匹配直到主串结束。

### 总结
KMP 的前缀计算（构建 `next` 数组）：
1.  **目的：** 预先计算模式串自身的“自匹配”信息（最长相等真前缀真后缀的长度），指导匹配失败时的跳转。
2.  **核心思想：** 利用已经计算好的 `next[0..i-1]` 的值，高效地计算 `next[i]`。当 `P[i]` 和 `P[j]` 不匹配时，不是简单地将 `j` 重置为0，而是根据 `next[j-1]` 将 `j` 回退到一个**可能匹配的位置**继续尝试。
3.  **时间复杂度：** 构建 `next` 数组的时间复杂度是 `O(m)`（虽然有两层循环，但 `j` 的回退总量是 `O(m)`）。整个 KMP 算法的时间复杂度是 `O(n + m)`。
4.  **关键价值：** 通过 `next` 数组，KMP 算法在匹配失败时能**避免主串指针 `i` 的回溯**，显著提高了匹配效率，尤其在模式串较长或包含较多重复片段时优势明显。

理解前缀计算（`next` 数组构建）是彻底掌握 KMP 算法的关键一步。它体现了“利用已知信息避免重复计算”的经典算法设计思想。
